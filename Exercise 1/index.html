Sections and general lines of an HTML5 document

The HTML5 specification brings many new elements for web developers. Allowing them to describe the web document structure with standarised semantics. This document describes these elements and how to use them to define the profile of any document.

Solved problems by HTML5

Both the structure definition of an HTML4 document and its profiling algorithm are very rough and tend to generate a number of problems:

  1. HTML5 removes the need of <div> elements to define semantic sections without needing to define specific values for the class attributes, adding a new element, <section>, the section element of HTML.
  2. Mixing different documents is tough: the inclusion of sub-documents in a main document. This is solved in HTML5 with sectioning such as: (<article>, <section>, <nav> and <aside>) they are always subsections of their closest parent section.
  3. HTML5 introduces the <hgroup> element that hides all heading elements except for the highest ranking one (For example <hgroup> <h1>Justine</h1><h2>Les Malheurs de la Vertu</h2> crea el perfil 1. Justine).
